ai 제안 방법
VirtualBox에서 USB 장치 목록을 불러오지 못하는 문제는 주로 Extension Pack 미설치, 사용자 권한 부족(Linux), 또는 USB 컨트롤러 설정 오류로 인해 발생합니다. Extension Pack을 설치하고, USB 2.0/3.0 컨트롤러를 설정한 뒤, 사용자를 vboxusers 그룹에 추가하여 해결할 수 있습니다. 
핵심 해결 방법
VirtualBox Extension Pack 설치 (필수): USB 2.0/3.0을 지원하려면 Oracle 홈페이지에서 다운로드하여 설치해야 합니다.
USB 컨트롤러 설정 변경: 가상 머신 설정 > USB에서 USB 2.0 (EHCI) 또는 USB 3.0 (xHCI) 컨트롤러를 선택합니다.
사용자 그룹 추가 (Linux): 터미널에서 sudo usermod -aG vboxusers $USER 명령어를 실행하고 로그아웃 후 다시 로그인합니다.
VBox USB 드라이버 재설치 (Windows): C:\Program Files\Oracle\VirtualBox\drivers\USB\filter 경로에서 VBoxUSB.inf 파일을 우클릭하여 설치합니다.
게스트 확장 설치: 가상 머신 내에서 VirtualBox Guest Additions를 설치합니다.

루트(/)용량이 부족할 때 1(home은 다른 파티션) : tmp 설정
.profile에
sudo mkdir /home/tmp
sudo chmod 777 /home/tmp
export TMPDIR=/home/tmp
export TMP=/home/tmp
export TEMP=/home/tmp

루트(/)용량이 부족할 때 2(home은 다른 파티션) : wine lib 옮기기
# 1. 대상 폴더를 다른 파티션으로 복사 (권한 유지 -p)
sudo cp -rp /opt/wine-stable /home/
# 2. 기존 폴더 이름 변경 (만약의 사태를 대비한 백업)
sudo mv /opt/wine-stable /opt/wine-stable_backup
# 3. 심볼릭 링크 생성
# ln -s [실제 데이터 위치] [원래 연결될 경로]
sudo ln -s /home/wine-stable /opt/wine-stable
# 4. 작동 확인 후 백업 삭제
sudo rm -rf /opt/wine-stable_backup

루트(/)용량이 부족할 때 3(home은 다른 파티션) : Bind Mount 활용
# 플랫팩 중단 확인 후 복사 (권한 보존을 위해 -a 사용)
sudo cp -av /var/lib/flatpak /mnt/data/
# 기존 폴더 백업 후 비우기
sudo mv /var/lib/flatpak /var/lib/flatpak_bak
sudo mkdir /var/lib/flatpak
# 임시 마운트 테스트
sudo mount --bind /mnt/data/flatpak /var/lib/flatpak
sudo systemctl start flatpak-system-helper.service
# 부팅 시 자동 적용 (/etc/fstab 수정)
/mnt/data/flatpak  /var/lib/flatpak  none  bind  0  0

# 플랫팩 중단 작업
flatpak ps
flatpak kill <ID 또는 이름>
sudo systemctl stop flatpak-system-helper.service
